# 개발 로드맵

기간은 1인 개발과 교육용 기록을 함께 수행하는 기준의 상대 추정치다. 구현보다 기록과 검증에 약 30~40%의 시간을 배정한다.

## 마일스톤

| 단계 | 목표 | 주요 결과물 | 예상 |
|---|---|---|---:|
| M0 | 프로젝트 기반 | Unity 프로젝트, Git, 폴더·패키지 규칙, 테스트 씬 | 1~2일 |
| M1 | 이동 샌드박스 | 캐릭터 이동, 카메라, 충돌, 입력 | 2~4일 |
| M2 | 1대1 전투 | 기본 공격, 체력, 피격, 사망, 데미지 표시 | 4~7일 |
| M3 | 적 AI | 탐지, 추적, 공격, 복귀, 다수 적 | 4~7일 |
| M4 | 스킬·성장 | 회피, 스킬 2종, 쿨다운, 경험치, 레벨업 | 4~7일 |
| M5 | 보스와 게임 흐름 | 보스 패턴, 승리·패배·재시작 | 4~7일 |
| M6 | 폴리싱 | 사운드, 이펙트, 카메라, 밸런스, 성능 | 5~10일 |
| M7 | 교육자료화 | 실습안, 실패 사례, 체크포인트, 최종 빌드 | 3~5일 |
